General remarks about raw data processing

  • In BioDare time 0 should coincides with the beginning of experimental conditions / ZT / subjective dusk/dawn.
    For that reason when raw data are uploaded time offset parameter can be used to adjust the time values accordingly. So for example, if the data were recorded from 12:00 o'clock and the subjective dawn was at 9:00, the first data point should have time of 3h, assuming that in raw data file first point is at 0, the time offset of 3h must be used during the raw data upload.
  • The timeseries described as Background noise, are treated as the background noise recording and they will be averaged and then subtracted from all the other time series during the raw data upload.
  • If some of the data should be ignored by BioDare they should be described as Ignored when importing.

Supported data formats

Excel table

Raw data are stored in Excel file, with one of the column holding the measurement time and the other columns recording measured values. The time column can represent either the actual time from start or as picture number which will be then converted to the time using the parameter: interval between images.

Details and constraints

  • Only the first data sheet is read from Excel file.
  • The time column should be located before the first data column (however, it does not have to be the first column in the sheet.
  • There may be missing values in timeseries, they are represented as blank cells in the sheet.
  • For time in hours, the time is represented as fraction of hour, so 1.25 means 1h 15minutes.
  • For time as image nr, the image numbers must be integers and starts from 1 (first picture taken), the images must be taken in equal intervals which are indicated by the time between images property.

TopCount

Raw data as produced by the TopCount reader for 96wells plates. TopCount can output data in many formats. We currently support:

  • plate map - interleaved blocks of measured values with blocks of measurement times. The blocks starts with columns numbers (1-12), the rows are indexed by letters (A-H).
  • column layout - 3 column file, first with well number, then measured value, then time stamp

Detail and constraints

  • The time can be represented either in AM/PM format (04:41:47 PM) or in 24h (17:13:58) and the consecutive dates are guessed by the transition from PM to AM
  • Single file can be constructed by concatenating (stacking) individual TopCount files.

TopCount Zip

Raw data consists of multiple TopCount files packed together in a zip file. Each sub file, represent data from on 96plate in same format as above.
